Poke-RL-0 Iterative Real-World Post-Training of Flow-Based VLAs
with Trust-Region Flow Matching Optimization.

An iterative post-training loop for flow-based vision-language-action policies. A unified TRFO objective turns demonstrations, human corrections, and autonomous rollouts — successes and failures alike — into a single, stable policy update, bounded by a frozen reference within a trust region.

Method

Method.

i · Value Expert
Value Expert · explainer Coming soon · 01:30
i. VALUE EXPERT

Sparse labels, dense value.

Each episode is annotated with a single binary success bit — no per-step rewards, no shaped signals, no demonstrator-dependent labels. From this minimal supervision, a distributional value expert learns a smooth curve that tracks remaining progress to task completion: rising as the policy approaches success, collapsing on the steps that precede failure.

The video on the left visualises one such learned value curve over a single episode.

ii · TRFO
TRFO objective · explainer Coming soon · 02:00
ii. TRFO

A single, unified objective.

TRFO collapses three previously distinct training regimes — imitation from demonstrations, on-policy reinforcement from rollouts, and safe negative learning from failures — into one weighted flow-matching loss.

$$\mathcal{L}_{\text{TRFO}}(\theta) \;=\; \mathbb{E}_{(o,a)\,\sim\,\mathcal{D}}\,\bigl[\,w(\hat{A})\cdot \ell_\theta(o,a)\,\bigr]$$

The signed weight w reduces the objective to standard flow matching when Â > 0, and to a trust-region-bounded repulsion when Â ≤ 0 — staying within the support of a frozen reference policy. One loss, one optimiser, one training pipeline; demos, corrections, and rollouts arrive on the same interface.

iii · Blending
Inter-chunk blending · explainer Coming soon · 01:10
iii. BLENDING

Smoother control, no extra training.

Inter-chunk blending is a training-free, inference-time procedure that absorbs network and model latency at the seams between consecutive action chunks.

The result is jerk-free, millimetre-precise control on the robot — exactly the substrate large-scale deployment and on-policy RL need to collect clean, faithfully on-policy data.
